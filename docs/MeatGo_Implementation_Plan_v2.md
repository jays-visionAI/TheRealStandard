## Part 1 — 시장 조사

### 1.1 시장 규모와 성장

| 지표 | 값 | 출처 |
|------|---|------|
| 국내 식자재 유통 시장 (2025) | **64조 원** (외식 42.4조 + 급식 11.7조 + 기타) | foodnews / foodtoday |
| 한국 콜드체인 물류 시장 | **약 USD 15B (~20조 원)**, CAGR 9.8% | Ken Research / Nexdigm |
| 빅3 (CJ프레시웨이/삼성웰스토리/현대그린푸드) 합산 매출 | 약 4조 (점유율 ~6%) | CJ뉴스룸 |
| 대기업 전체 점유율 | **약 15%** | 헤럴드경제 |
| **영세·중소 유통업체 점유율** | **85%+** ← 디지털화 미진 | 머니투데이 |

→ **시사점**: 시장 자체가 매우 크고, 80% 이상이 비효율적 영세 유통업체임. **이게 MeatGo의 진짜 기회.**

### 1.2 주요 플레이어 지도

```
[ B2C 새벽배송 ]          [ B2B 식자재 (대기업) ]        [ B2B SaaS / 마켓플레이스 ]
─────────────────         ─────────────────────         ─────────────────────────
마켓컬리                    CJ프레시웨이 (1위, 3.5조)       마켓봄 (마켓보로) — SaaS
쿠팡 로켓프레시              삼성웰스토리                    식봄 (마켓보로) — 마켓플레이스
오아시스마켓                현대그린푸드                    미트박스 — 축산 전문 (누적 1조)
정육각 (D2C 정육)            동원홈푸드, 풀무원푸드머스
                          신세계푸드
```

**MeatGo가 들어가는 자리**:
- B2C 새벽배송 격전지 (마켓컬리/쿠팡/오아시스) **회피**
- B2B 식자재 빅3는 대형 외식 체인 위주 ← 영세-중형은 무관심
- 미트박스/마켓봄은 **마켓플레이스 모델** ← MeatGo는 **직접 유통 + 디지털 인프라** 하이브리드

### 1.3 디지털 전환 트렌드

- **IoT 콜드체인 모니터링** — 차량/창고 온도 실시간 추적 (대기업 도입 중)
- **AI 수요예측** — CJ프레시웨이 400억 투자
- **축산물 이력추적 의무화** — 개체번호 추적 강화 (정부 정책)
- **새벽배송의 B2B 확산** — 식당 새벽 입고 수요 ↑
- **자동발주** — POS 연동 자동 재주문 (체인 외식)

> ⚠️ 블록체인 기반 이력추적은 본 사업의 초기 12개월 로드맵 **범위 외**. 일반 DB 기반 이력관리로 충분.

### 1.4 고객 페인포인트

| 페르소나 | 핵심 페인포인트 |
|---------|----------------|
| **개인 식당 사장** | 가격 변동성 큼 · 품질 들쭉날쭉 · 외상 운영 정산 복잡 · 견적 받기 번거로움 |
| **중형 외식 체인 (지점 10~50개)** | 다점포 발주 통합 어려움 · 메뉴 단위 발주 안 됨 · 수령 검수 표준화 안 됨 |

## Part 2 — Strategic Positioning

### 2.1 비전 선언

> **"육류에서 시작해, 국내 최대의 데이터 기반 신선식품 B2B 유통 인프라로."**
>
> 영세·중형 거래처가 자본/IT 없이도 빅3 수준의 발주·물류·정산을 누리도록.

### 2.2 차별화 전략 — 3축

**① 데이터 깊이 (Data Depth)**
빅3가 못 하는 **거래처별 수익성, 품목별 가격 트렌드, 매수 타이밍 AI** 를 영세 거래처에게 제공.

**② 양면(Two-sided) 인프라**
- 공급자 측: 농가/도축장이 MeatGo로 일괄 출고 → 수요 예측 데이터 제공
- 구매자 측: 식당이 MeatGo에서 일괄 발주 → 정산/외상/할인 통합
- 양측 거래 데이터 축적 → 가격 발견(price discovery) 정확도 ↑

**③ 카테고리 점진적 확장**
**Year 1: 육류** → **Year 2: 수산** → **Year 3: 채소/과일** → **Year 4: 양념/가공식품**
공통 인프라(콜드체인, 정산, 카탈로그, 발주) 재사용으로 확장 비용 ↓.

### 2.3 12개월 확장 비전

```
M+1 ~ M+2    Phase 0 — MVP 정합성 마무리
M+2 ~ M+4    Phase 1 — 카탈로그/콘텐츠 풍부화 (런칭 콘텐츠)
M+4 ~ M+5    Phase 2 — 운영 효율화 + 영업 도구
M+5 ~ M+7    Phase 3 — AI 인사이트 v1
M+7 ~ M+10   Phase 4 — 수산 카테고리 확장
M+10 ~ M+12  Phase 5 — 마켓플레이스화 (공급자 셀프 등록)
```

### 2.4 핵심 KPI

| KPI | M+3 목표 | M+6 목표 | M+12 목표 |
|-----|----------|----------|-----------|
| 활성 거래처 (월 발주 ≥1회) | 10 | 50 | 200 |
| 누적 거래액 | ₩1억 | ₩10억 | ₩50억 |
| 월 활성 공급사 | 3 | 10 | 30 |
| 평균 발주처리 시간 | <15분 | <10분 | <5분 |
| 정산 자동화율 | 70% | 90% | 95% |

## Future Track — 체인사업자 AX 솔루션

> 초기 12개월 로드맵 **범위 외**. M+12 이후 진행 가능성 검토.

### 개념

중형 외식 체인(지점 10~100개)이 MeatGo의 발주/정산/POS 연동 인프라를 **화이트라벨**로 사용하면서 동시에 MeatGo로부터 식자재를 공급받는 **양면 모델**.

### 가치 제안

- **체인사업자에게**: 자체 IT 구축 없이 디지털 발주/정산/이력관리 인프라 즉시 이용
- **MeatGo에게**: 거래 락인(lock-in) 강화 + 안정적 매출 + 데이터 축적
- **공급자에게**: 체인 단위 대량 수요 가시화로 생산 예측 정확도 ↑

### 검토 시점

- M+12에 거래처 200개 이상 확보 후
- 초기 체인 3~5개 PoC → M+15까지 정식 베타
- 정식 출시는 M+18 이후 (Year 2 후반)

### 잠재 차별화 포인트

- **체인 본사용 BI** — 전 지점 식자재 사용량/원가율 통합 대시보드
- **지점별 자동발주** — POS 매출 → 자동 식재 발주 계산
- **체인 표준 레시피 카드** — 메뉴별 원가 계산 자동화
- **분점 출점 자문** — 입지별 식자재 단가/물류 비용 시뮬레이션
