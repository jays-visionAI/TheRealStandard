# 고객 인사이트 모듈 — 설계 스펙

**작성일**: 2026-06-10
**대상 화면**: 고객 대시보드(`/order/dashboard`)
**원칙**: 현재 데이터(희소)로 **정직하게 동작하는 것만** 구현. ML/예측 아님 — 룰 기반 + 표본 가드. LLM 해설은 후순위 옵션 레이어.

---

## 1. 핵심 분기 — 사용자 상태에 따라 다른 인사이트

```
로그인 고객
 ├─ 본인 주문 0건  →  [COHORT 모드]   플랫폼 평균/인기 데이터 ("비슷한 거래처들은…")
 └─ 본인 주문 ≥1건 →  [PERSONAL 모드] 본인 이력 기반 맞춤 분석 (+코호트 교차추천)
```

- 랜딩의 "AI 의사결정 브리핑"은 **마케팅 예시(목업)**. 실제 가치는 이 모듈이 로그인 후 제공.
- 모든 카드에 **데이터 출처 라벨**(`내 주문 기반` / `플랫폼 평균`)을 표기해 과장 없이 신뢰 확보.

## 2. 인사이트 정의 (v1 — 룰 기반)

### PERSONAL 모드 (본인 주문 ≥1건)
| kind | 이름 | 룰 | 발화 조건 |
|------|------|-----|----------|
| `reorder` | 재주문 알림 | 품목별 주문일 간격 평균 → 마지막 주문 후 경과일 ≥ 평균주기×0.9 | 동일 품목 주문 ≥2회 |
| `spend` | 지출 트렌드 | 이번 달 vs 지난 달 주문금액 ±% | 지난 달 >0 |
| `top` | 단골 품목 | 본인 품목별 중량/금액 합계 Top 3 | 품목 ≥1 |
| `cross` | 교차 추천 | 플랫폼 인기 품목 − 본인 구매 품목 = 미구매 인기품 Top 2 | 코호트 데이터 존재 |
| `settlement` | 정산 임박/연체 | 미수 잔액 >0 → 가장 가까운 결제기한 D-day (경과 시 warning) | 미수 존재 |

### COHORT 모드 (본인 주문 0건)
| kind | 이름 | 룰 |
|------|------|-----|
| `popular` | 인기 품목 | 전체 salesOrderItems 중량 기준 Top 5 |
| `pattern` | 거래 패턴 | 최근 30일: 활성 거래처 수·발주 건수·건당 평균 금액 |
| `start` | 시작 안내 | 첫 주문 CTA (카탈로그) |

## 3. 신뢰 가드 (정직성 장치)
- **표본 가드**: 전체 주문 <5건이면 "플랫폼 초기 데이터 기준 참고용" 캡션 표기.
- **개별 폴백**: PERSONAL 모드라도 조건 미달 인사이트는 생략(빈 카드 금지). 카드 0개면 코호트 카드로 채움.
- **출처 라벨 필수**: personal/cohort 배지.
- 집계 실패(권한/네트워크)는 카드 생략 — 대시보드를 깨지 않음.

## 4. 데이터 소스 & 권한 (모두 기존 인프라)
| 데이터 | 함수 | 권한 |
|--------|------|------|
| 본인 주문 | `getSalesOrdersByCustomer` | 인증 ✅ |
| 코호트 주문 | `getAllSalesOrders` | 인증 ✅ |
| 품목 | `getAllSalesOrderItems` | 인증 ✅ |
| 본인 정산 | `getSettlementsByCustomer` | 본인 ✅ |

- v1은 클라이언트 집계(현재 데이터 규모에서 충분). 데이터 증가 시 `publicStats` 서버 집계로 이전(성능+개인정보).

## 5. 로드맵
- **v1 (지금)**: 위 룰 엔진 + 대시보드 카드. 외부 의존 0.
- **v2**: Claude 해설 레이어 — 룰 결과를 자연어 요약(키 필요, 옵션). 시세(EKAPE/KAMIS) 연계 "내 단골 품목 시세 변동".
- **v3**: 수요예측·재주문 자동제안(orderTemplates 연계), publicStats 서버 집계.
