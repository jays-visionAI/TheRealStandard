# MeatGo — AI 의사결정 시스템 API 연동 리스트

**작성일**: 2026-06-09
**목적**: AI 기반 의사결정(매수타이밍·가격책정·수요예측·재고·거래처·물류) 구축에 필요한 외부 API 연동 로드맵
**현황 표기**: ✅ 이미 코드 연동(키만 입력) · 🆕 신규 연동 필요 · ⚙️ 자체 구축 권장

> ⚠️ 본 문서의 엔드포인트/제공처는 연동 설계 기준이며, 실제 신청 시 각 기관의 최신 API 명세·요금을 확인할 것.

---

## 0. 시스템 개요 — 데이터 → 추론 → 의사결정 → 실행

```
[입력 데이터]                      [AI 추론]              [의사결정]            [실행]
국내 시세(경락/도매) ┐
수입육 단가/관세    │
사료·곡물·환율·유가  ├──▶  LLM(Claude) +  ──▶  매수타이밍       ──▶  발주 자동제안
도축두수/수급       │      자체 시계열모델       가격 책정             단가표 갱신
날씨/계절           │                            재고 처분             알림톡/이메일
자체 거래데이터 ────┤                            거래처 이탈대응        세금계산서
뉴스 감성          ┘                            공급사 평가           배차 최적화
```
- **자체 거래 데이터**(salesOrders·settlements·inventory·priceHistory)는 이미 보유 → AI 입력으로 즉시 사용 가능.
- **외부 API**는 위 데이터에 "시장 맥락"을 더해 예측 정확도를 높임.
- 모든 외부 호출은 **Cloudflare Worker 게이트웨이 경유**(CORS·비밀키 서버측 처리, 이미 구축됨).

## 우선순위 통합 (구축 순서 권고)

| 순위 | 묶음 | 포함 API | 효과 |
|------|------|----------|------|
| **1** | **AI 엔진 + 보유 시세 점화** | Claude API + EKAPE·KAMIS(키 입력) + 네이버뉴스 | 자체 거래데이터 + 시세 + 뉴스만으로 **1차 인사이트(매수타이밍·수익성)** 즉시 가능 |
| **2** | **원가 선행지표** | 관세청 수입육 · ECOS 환율 · 오피넷 유가 · 사료/곡물 | 매수타이밍·원가예측 정확도↑ |
| **3** | **수급·환경 예측** | 등급판정 · 도축두수 · 기상청 | 수요·공급 예측, 재고 의사결정 |
| **4** | **KYC·이력·신용** | 국세청 사업자 · 축산물이력 | 거래처/공급사 자동 검증·신뢰 스코어 |
| **5** | **물류·실행** | 카카오모빌리티 · 솔라피 · Resend · 팝빌 | 의사결정→자동 실행(배차·알림·발행) |

> **1순위만으로도 "AI 의사결정 v1"이 동작**합니다 — 이미 보유한 거래데이터 + 점화만 하면 되는 시세/뉴스 + Claude. 나머지는 정확도·자동화를 단계적으로 강화.

## 아키텍처 권고

1. **게이트웨이 경유 통일**: 모든 외부 API는 Cloudflare Worker(`/api/*`)로 프록시 → 비밀키 서버측, CORS 해결(이미 구축). 키는 Worker secret 또는 Firestore `system/settings`.
2. **수집 배치**: Worker Cron(또는 Cloud Functions)으로 시세·뉴스·거시지표를 **일 1회 수집 → `marketPrices`/시계열 적재** → AI는 적재 데이터를 읽음(매 호출 외부 API 직접 호출 회피, 비용·속도↑).
3. **AI 호출 구조**: 적재 데이터 + 자체 거래집계(`profitabilityService` 등)를 컨텍스트로 묶어 Claude에 전달 → 구조화 출력(추천·근거·신뢰도) → `notifications`/대시보드 노출.
4. **비용 통제**: 외부 시세 API는 대부분 무료(공공데이터). 유료는 Claude(토큰)·카카오모빌리티(사용량)·신용평가(계약)·SaaS(알림톡/이메일/팝빌). 캐싱·배치로 최소화.
5. **데이터 거버넌스**: 외부 데이터 출처·수집시각을 함께 저장(감사·재현성), 개인정보 포함 데이터(거래처)는 집계·마스킹.
