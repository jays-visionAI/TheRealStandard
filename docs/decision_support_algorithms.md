# MeatGo — AI 의사결정 지원 알고리즘 설계

**작성일**: 2026-06-09
**목적**: 육류유통 핵심 의사결정 6종에 대한 구체 알고리즘(수식·신호·가중치·임계값) + 룰/LLM 하이브리드 구조
**연관**: `docs/ai_decision_api_integration.md`(데이터 소스), `docs/technical_due_diligence.md`

> 설계 철학: **룰 기반 점수화로 투명·재현 가능한 신호를 만들고, LLM(Claude)이 다신호를 종합해 근거·신뢰도·예외를 설명**한다. 위험 의사결정은 항상 사람 승인(human-in-the-loop).

## 0. 공통 아키텍처

```
[L1 데이터]         [L2 피처/신호]            [L3 점수엔진(룰)]      [L4 추론(LLM)]        [L5 실행]
자체 거래데이터  →  정규화·시계열·집계   →   신호별 0~1 점수   →   Claude 종합·설명  →  추천카드
외부 시세/거시       (z-score, 분위수,        가중합 → 0~100        + 신뢰도/예외       (승인→발주/가격/알림)
                     이동평균, RFM)           임계값 분류
```

### 0.1 공통 수식 도구
- **분위수 위치** `pct(x, series)` = series에서 x의 백분위(0~1).
- **z-score** `z(x) = (x − μ) / σ` (최근 N일 기준).
- **정규화** `norm(x, lo, hi)` = clamp((x−lo)/(hi−lo), 0, 1).
- **추세 기울기** `slope` = 최근 k점 선형회귀 기울기 ÷ 평균값(상대화).
- **지수가중이동평균** `EWMA_t = α·x_t + (1−α)·EWMA_{t−1}`.

### 0.2 신뢰도(confidence)
모든 추천은 `confidence ∈ {높음, 중간, 낮음}`. 결정 요인:
- 데이터 충분성(표본 수·기간), 신호 일치도(신호들이 같은 방향?), 외부데이터 신선도(수집 경과시간).
- `confidence = f(표본충분성, 신호일치도, 데이터신선도)` — 셋 중 둘 이상 미달 시 "낮음" + 사람검토 강제.

## 1. 매수 타이밍 추천 (Buy Timing)

**목적**: 공급사로부터 **언제·얼마나 매입**할지. "이번 주 매수 적기" 판단.

### 입력 신호 (각 0~1, 높을수록 매수 유리)
| 신호 | 정의 | 데이터 |
|------|------|--------|
| `S_price` 가격 위치 | `1 − pct(현재경락가, 최근60일)` (쌀수록↑) | EKAPE 경락가 |
| `S_dip` 저점 반등 | 최근 하락 후 반등 초입이면↑ (slope 음→양 전환) | priceHistory |
| `S_cost` 원가 상승 압력 | 사료·환율·유가 상승추세 → 미리 매수 유리 | ECOS·오피넷·곡물 |
| `S_supply` 공급 축소 | 도축두수 감소추세 → 가격 상승 예상 → 선매수 | 도축두수 |
| `S_demand` 수요 임박 | 명절/계절 수요 급증 D-14 이내 | 캘린더·기상청 |
| `S_inv` 재고 필요 | `1 − norm(현재고일수, 0, 목표일수)` (부족할수록↑) | inventory |
| `S_senti` 뉴스 감성 | 공급차질·가격상승 뉴스 → ↑ (Claude 분류) | 네이버뉴스+Claude |

### 점수
```
BuyScore = 100 × Σ(wᵢ · Sᵢ),  Σwᵢ = 1
기본 가중치: price .25 · dip .10 · cost .15 · supply .15 · demand .10 · inv .15 · senti .10
```
**분류**
- `≥ 75` **강력 매수** (목표 재고 상한까지 확보 제안)
- `55–75` **분할 매수** (필요량 50%)
- `40–55` **관망**
- `< 40` **보류** (재고 소진 우선)

### 출력
`{ 추천: 강력매수, 점수: 78, 제안수량: N박스, 근거: [경락가 60일 최저 12%, 환율↑로 수입원가 상승 예상, 명절 D-10], 신뢰도: 높음 }`

### LLM(Claude) 역할
신호 점수 + 원시 데이터를 받아 **① 자연어 근거 요약 ② 상충 신호 해설(예: 가격 낮지만 수요 약함) ③ 리스크 경고 ④ 신뢰도 보정**. 자동 발주는 금지 — 사람 승인.

## 2. 판매 가격 책정 (Dynamic Pricing)

**목적**: 상품·거래처별 **공급 단가 제안**. 원가·시장·거래처가치·재고 동시 반영.

### 알고리즘
```
basePrice   = costPrice × (1 + targetMargin)            // 원가 + 목표마진
marketPrice = KAMIS도매가 × marketAdj                    // 시장 기준
floorPrice  = costPrice × (1 + minMargin)               // 마진 하한(가드)

tierMult    = 거래처등급 배수 (수익성/물량/결제조건 기반)
              CM상위·대량·선결제 → 0.97 (우대) / CM하위·소량·외상 → 1.05
invAdj      = 과잉재고 → 0.95 / 부족 → 1.03 / 정상 → 1.00
mktTrendAdj = 시세 상승추세 → 1.0~1.04 / 하락 → 0.97~1.0

suggested   = max(floorPrice, mix(basePrice, marketPrice) × tierMult × invAdj × mktTrendAdj)
              (mix = 원가가중 0.6 · 시장가중 0.4, 설정 가능)
```
**거래처 등급 배수**는 `profitabilityService`의 공헌이익 순위 + `paymentTermDays` + 월 물량으로 산출.

### 출력
`{ 상품, 거래처, 제안단가, 현재단가, Δ, 예상마진%, 근거: [원가+15%, 도매가 대비 −3%, A등급 우대, 재고 과잉 −5%], 신뢰도 }`
- **가드레일**: `floorPrice` 하회 불가, 단가 변동 ±X% 초과 시 사람 승인.

### LLM 역할
거래처 협상 맥락(이탈위험·경쟁사 단가·관계) 반영한 **권장 단가 밴드 + 협상 멘트** 생성. 최종 확정은 영업.

## 3. 재고 처분 추천 (Inventory Liquidation)

**목적**: 신선/유통기한 리스크 재고를 **정상판매 / 할인 / 도매덤핑 / 폐기경고**로 분류.

### 신호
```
ageDays        = 입고 후 경과일
shelfRatio     = ageDays / 유통기한일                    // 1에 가까울수록 위험
velocity       = 최근 회전율(EWMA 일평균 출고 ÷ 현재고)   // 낮을수록 위험
daysToClear    = 현재고 ÷ max(velocity, ε)               // 소진예상일
priceTrend     = 해당 부위 시세 추세(하락 시 보유손실↑)
```
### 긴급도
```
Urgency = 100 × clamp( 0.5·shelfRatio + 0.3·norm(daysToClear, 7, 30) + 0.2·max(0,−priceSlope), 0, 1 )
```
**분류**
- `daysToClear > 유통기한 잔여일` → **🔴 폐기 위험 → 즉시 덤핑/할인**
- `Urgency ≥ 70` → **할인 판매**(추천 할인율 = f(긴급도))
- `40–70` → **프로모션/끼워팔기**
- `< 40` → **정상 판매**

### 출력
`{ 상품, 현재고, 소진예상 22일 > 잔여 9일, 추천: 도매덤핑, 권장할인 18%, 근거, 신뢰도 }`

### LLM 역할
처분 채널 추천(특정 거래처·도매시장), 묶음 프로모션 아이디어, 손실 최소화 시나리오 비교.

## 4. 수요 예측 (Demand Forecast)

**목적**: 상품·거래처별 **다음 주/기간 예상 발주량** → 매입·재고·자동발주 입력.

### 알고리즘 (단순·견고 우선)
```
baseline   = EWMA(주간 발주량, α=0.4)                    // 추세 반영 이동평균
seasonal   = 요일·월·명절 계절지수 (과거 동월/동요일 비율)
holidayAdj = 명절/연휴 보정계수 (캘린더)
weatherAdj = 폭염·한파 등 수요탄력 (선택, 기상청)

forecast   = baseline × seasonal × holidayAdj × weatherAdj
구간        = forecast ± z·σ_resid (예측 신뢰구간)
```
- 표본 부족(< 8주) 상품은 **동일 카테고리 평균으로 대체** + 신뢰도 "낮음".
- 고도화 시 Holt-Winters/Prophet 또는 자체 ML(⚙️)로 교체 가능.

### 출력
`{ 상품, 다음주 예상 320kg (250~390), 추세 ↑, 근거: 4주평균 290 + 명절보정 +12%, 신뢰도: 중간 }`

### LLM 역할
이상치 해석(일회성 대량발주 제외), 신규 상품 콜드스타트 추론, 예측 가정 설명.

## 5. 거래처 이탈 예측 (Churn Risk)

**목적**: 발주 감소 패턴을 조기 감지 → **이탈 위험 알림 + 영업 액션**.

### RFM + 추세 신호
```
R 최근성  = 마지막 발주 경과일 / 평소 발주간격          // >2 면 경고
F 빈도추세 = 최근4주 발주횟수 / 직전4주                  // <0.6 면 감소
M 금액추세 = 최근4주 금액 / 직전4주                      // <0.7 면 감소
claims    = 최근 클레임/반품 건수
overdue   = 미수 연체일수
```
### 위험점수
```
Churn = 100 × clamp( 0.35·norm(R, 1, 3) + 0.25·(1−F) + 0.25·(1−M) + 0.10·norm(claims,0,3) + 0.05·norm(overdue,0,30), 0, 1 )
```
**분류**: `≥70` 고위험(즉시 영업 콜) · `45–70` 주의(프로모션/단가검토) · `<45` 안정.

### 출력
`{ 거래처, 위험 76, 신호: [마지막발주 18일(평소 6일), 금액 −40%, 클레임 2건], 추천: 담당 직접 연락 + 단가 재협상, 신뢰도 }`

### LLM 역할
이탈 원인 가설(가격/품질/경쟁사), 맞춤 리텐션 제안(특가·결제조건 완화), 우선순위 정렬.

## 6. 공급사 신뢰도 스코어 (Supplier Score)

**목적**: 공급사 종합 점수로 **발주 배분·신규 입점 심사** 지원.

### 지표 (각 0~1)
```
delivery  = 1 − 입고지연율(약속일 대비)
quality   = 1 − 품질이슈율(클레임/반품 ÷ 납품건)
priceStab = 1 − 단가 변동성(σ/μ)                         // 안정적일수록↑
traceability = 이력정보 충실도(개체번호·HACCP 제출률)
settle    = 정산 정확도(분쟁 없는 비율)
capacity  = 공급능력 충족도(요청 대비 납품)
```
### 종합
```
SupplierScore = 100 × (0.25·delivery + 0.25·quality + 0.15·priceStab + 0.15·traceability + 0.10·settle + 0.10·capacity)
```
**등급**: A `≥85` · B `70–85` · C `55–70` · D `<55`(개선요구/거래축소).
→ 발주 시 **동일 품목 다공급사 비교**(점수·단가·거리)로 자동 배분 제안(Phase 5 매칭엔진의 입력).

### LLM 역할
점수 저하 원인 진단, 공급사 피드백 코칭 멘트, 리스크 분산 추천.

## 7. 통합 — "오늘의 의사결정 브리핑"

매일 배치가 위 6종을 산출 → 관리자 대시보드 1화면:
```
🌅 오늘의 의사결정 (2026-06-09)
• 매수: 삼겹살 강력매수(78) — 60일 최저가 + 환율↑   [발주 제안 보기]
• 가격: 목심 단가 −3% 권장(재고 과잉)              [단가표 반영]
• 재고: 등심 9일내 소진필요 → 할인 18% 추천         [프로모션]
• 이탈: ○○식당 위험76 → 담당 연락                  [리텐션]
• 공급사: △△미트 B→C 하락(지연율↑)                [점검]
```
- 각 항목 = 룰 점수 + Claude 근거 + 사람 승인 버튼.

## 8. 구현 매핑 & 가드레일

| 알고리즘 | 신규 서비스 | 입력(기존) | 외부 |
|----------|-------------|------------|------|
| 매수타이밍 | `buyTimingService` | priceHistory·inventory | EKAPE·ECOS·도축두수·뉴스 |
| 가격책정 | `pricingService` | products·profitabilityService | KAMIS |
| 재고처분 | `liquidationService` | inventory·products | EKAPE |
| 수요예측 | `demandForecastService` | salesOrders·orderSheets | 캘린더·기상청 |
| 이탈예측 | `churnService` | salesOrders·settlements | — |
| 공급사스코어 | `supplierScoreService` | purchaseOrders·shipments | 이력제 |
| 종합/LLM | `aiAdvisorService` | 위 전체 | Claude API |

**가드레일(필수)**
1. **사람 승인**: 발주·가격·세금계산서 등 금전·외부영향 의사결정은 자동 실행 금지.
2. **설명가능성**: 모든 추천에 신호별 기여도·원시값 노출(블랙박스 금지).
3. **신뢰도 게이팅**: confidence 낮음 → 추천 보류 또는 "참고용" 표기.
4. **데이터 신선도 체크**: 외부 데이터 N시간 초과 미갱신 시 해당 신호 가중치 0 + 경고.
5. **편향/이상치 방어**: 일회성 대량발주·결측·키 미입력은 제외하고 계산, 로그.
6. **감사 로그**: 추천 시점 입력 스냅샷 저장(사후 검증·모델 개선).

## 9. 단계적 도입

| 단계 | 범위 | 외부 의존 |
|------|------|-----------|
| **v1 (룰 기반)** | 이탈예측·수요예측·재고처분 — 자체 데이터만 | 없음 (즉시 가능) |
| **v2 (시세 결합)** | 매수타이밍·가격책정 + LLM 근거 | EKAPE·KAMIS 키 + Claude |
| **v3 (거시·예측 고도화)** | 원가 선행지표·계절·날씨 + 신뢰구간 | ECOS·오피넷·기상청·도축두수 |
| **v4 (자동화·매칭)** | 공급사 매칭·자동발주 제안·실행연동 | 이력제·알림톡·배차 |

> **v1은 외부 키 없이 자체 거래데이터만으로 즉시 구축 가능** — 이탈·수요·재고처분 룰 엔진부터 시작 권장.

---

*본 설계는 투명성(룰)·설명가능성(LLM 근거)·안전성(사람 승인)을 원칙으로 한다. 가중치·임계값은 운영 데이터로 백테스트하며 보정한다.*
