# TRS (The Real Standard) 주문 시스템 워크플로우 가이드

본 문서는 TRS 시스템의 주문 유형별 상세 워크플로우를 안내합니다. 모든 프로세스는 **'발주서'**라는 단일 표준 문서를 통해 유기적으로 연결됩니다.

---

## 1. 정식 거래처(회원) 프로세스: **발주 제안형**
정적 거래처의 구매 편의를 높이고, 관리자가 특정 품목을 타겟팅하여 주문을 유도할 때 효율적입니다.

1.  **발주서 생성 및 발송 (관리자)**
    *   관리자가 정식 거래처를 선택하고, 해당 거래처가 주로 구매하는 품목들을 담은 발주서를 생성합니다.
    *   생성된 발주서의 **고객 전용 링크**를 복사하여 전송합니다.
2.  **수량 입력 및 주문 제출 (고객사)**
    *   고객은 로그인 후 링크를 통해 접속하거나 마이페이지에서 발주서를 확인합니다.
    *   제안된 품목 중 필요한 수량을 입력하고 '주문 확인 및 제출'을 클릭합니다. (**상태: 제출완료**)
3.  **검토 및 최종 승인 (관리자)**
    *   관리자는 고객이 제출한 수량을 확인하고, 재고 상황에 따라 최종 조정 후 '확정'합니다. (**상태: 승인완료**)
4.  **물류/매입 연동**
    *   승인된 데이터는 즉시 매입발주 및 창고 출고 프로세스로 자동 연동됩니다.

---

## 2. 비회원/신규 고객 프로세스: **단가표 기반 전환형**
단가표(견적서)를 먼저 배포하고, 이를 확인한 잠재 고객이 자연스럽게 주문으로 이어지는 방식입니다.

1.  **단가표 링크 배포 (관리자)**
    *   관리자가 범용적으로 사용할 수 있는 공용 단가표 링크를 생성하여 잠재 고객에게 공유합니다.
2.  **즉시 주문 및 정보 입력 (비회원)**
    *   비회원 고객이 단가표에서 [즉시 주문하기]를 클릭하고, 업체명과 연락처를 입력합니다.
3.  **임시 발주서 자동 생성 (시스템)**
    *   입력된 정보를 바탕으로 **해당 고객만을 위한 임시 발주서가 즉시 생성**됩니다.
    *   고객은 로그인 없이 이 화면에서 바로 주문 수량을 입력하고 제출할 수 있습니다.
4.  **정식 거래처 유도 (관리자)**
    *   제출 완료 화면에서 **"정식 거래처 신청(회원가입)"**을 유도하여 지속적인 거래 관계를 형성합니다.
    *   관리자는 수정된 내용을 다시 링크로 만들어 전달하여 고객의 최종 확정을 받습니다.

---

## 3. 주문 상태별 진행 단계

| 상태 (영문) | 상태 (국문) | 설명 | 다음 행동 |
|:--- |:--- |:--- |:--- |
| **DRAFT / SENT** | **초안 / 발송됨** | 관리자가 주문 내용을 꾸리거나 제안한 상태 | 고객의 수량 입력 대기 |
| **SUBMITTED** | **제출완료** | 고객이 주문 수량을 결정하여 컨펌을 요청한 상태 | 관리자의 최종 검토 및 승인 |
| **REVISION** | **수정요청** | 단가나 수량의 재협의가 필요한 상태 | 고객의 재확인 또는 관리자 재승인 |
| **CONFIRMED** | **승인완료** | **모든 합의 완료 및 주문 확정** | 매입/배차/출고 프로세스 실행 |

---

## 4. 시스템의 특장점

*   **주문 가드레일 제공**: 고객이 상품을 찾는 수고를 덜어주도록 관리자가 미리 발주 품목을 세팅하여 제공합니다.
*   **끊김 없는 전환**: 견적(단가표) 확인부터 실제 발주(발주서 작성)까지 막힘없이 이어지는 사용자 경험을 제공합니다.
*   **표준화된 관리**: 회원, 비회원 가리지 않고 모든 주문이 '발주서' 데이터로 통합되어 관리 업무가 단순해집니다.
